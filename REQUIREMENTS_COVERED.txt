Project: Hospital Appointment Booking System

Implemented Roles:
1. Patient
2. Doctor
3. Receptionist
4. Admin

Shared database tables included:
users, patients, specializations, doctors, doctor_availability, leave_dates, appointments,
consultation_notes, billing, doctor_reviews, dependents, announcements.
Extra support tables: hospital_settings, complaints, patient_messages.

Technical requirements covered:
- MVC folder pattern: model, view, controller
- Model folder is separated like controller: user, patient, doctor, receptionist, admin
- PHP + MySQL
- mysqli prepared statements are used through BaseModel.php
- SQL queries are kept basic: simple SELECT / INSERT / UPDATE / DELETE with placeholders
- No raw string insertion is used in database query values
- PHP sessions for authentication
- Role-based access on protected pages using helper/auth.php
- AJAX per role with JSON API
- Basic server-side validation in controllers
- Simple CSS only

Patient features covered:
Registration, login/logout, profile update, profile picture upload, password change, emergency contact,
medical history notes, dependents add/edit/delete, browse/search/filter doctors, doctor details, AJAX slots,
appointment booking for self/dependent, upcoming appointments, cancel/reschedule, past history,
consultation notes, billing history, payment intent, doctor review add/edit/delete, announcements.

Doctor features covered:
Login, profile management, specialization/fee/license/experience/photo update, weekly availability,
leave dates, today schedule, weekly calendar, confirm/reject/no-show, AJAX check-in, completed appointment notes,
patient notes history, reviews and reply, earnings, statistics, follow-up list, patient messages and reply.

Receptionist features covered:
Login, daily appointment schedule, filter by status, patient search, register patient, walk-in booking,
AJAX slot check, check-in, waiting queue, payment processing, printable receipt, cancel appointment,
doctor availability by slots, daily summary report.

Admin features covered:
Dashboard stats by AJAX, doctor account add/edit/approve/deactivate, specializations CRUD,
receptionist create/edit/deactivate, patient search/deactivate, appointment policies, all appointment filters,
revenue report, volume report, performance report, complaints resolve, announcements, billing dashboard,
printable/export HTML tables.
